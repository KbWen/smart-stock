# /test

Design minimal necessary tests per `docs/TESTING_PROTOCOL.md`.

Output Format:

1. Test list
2. Corresponding commands or test scripts
3. Test results and known limitations

## Test Persistence (Hard Gate)

- Test code MUST be written to the project's test directory (e.g., `tests/`, `__tests__/`, or project convention).
- Test files follow naming from `docs/TESTING_PROTOCOL.md`.
- Work Log MUST record: `Test Files: [list of test file paths]`.
- Evidence: test output (pass/fail) MUST be logged in Work Log, not just claimed.
